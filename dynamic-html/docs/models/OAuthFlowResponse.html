OAuthFlowResponse

  • access_token : String
    Contains a JWT token that contains fields like exp (expiration timestamp), iat (issued at), or usr (which represent the granted user account). Expiration timestamp (UTC seconds from 1970) can be used to know when it is required to refresh the access token, which usually have a life span of some hours.
  • expires_in : BigDecimal
    Number of seconds while the provided access token will be valid in the server.
  • resfresh_token : String
    Contains a JWT token that contains fields like exp (expiration timestamp), iat (issued at), or jti (internal token identifier). Refresh tokens can be used to both obtain new access and refresh tokens over the OauthRefreshTokenFlow. Expiration timestamp (UTC seconds from 1970) provides a timestamp that indicates when the token will become invaild, so it could not be used for getting new access or refresh tokens. It usually have a life span of some months, if not invalidated in the server.
  • scope : String
    By default it will be always null.
  • token_type : String
    By default it will be always bearer. It means that the provided access token should be included in the Authorization header. It can be used alternativelly as a request parameter called 'autorization'
